# LINE 記帳小工具

- 只要有 LINE 跟一個 Google 帳號，就能零負擔地記錄個股交易明細，不需要學任何記帳 App。
- 用 LINE 打字記帳，或直接在自己的 Google 試算表填資料都可以，兩邊資料同步。

> ✅ **目前狀態：Phase 1 MVP 已完成並上線。** 
核心記帳、損益引擎、OAuth 連結、查詢／刪除／同步、LIFF 儀表板皆已完成並部署於 Google Cloud Run。

## 想解決的問題

- **看盤 App 解決「即時報價」，沒有解決「多人、多帳戶、長期記帳」**：券商 App 看得到自己在那家券商的庫存損益，但看不到「我在三家券商加起來總共持有多少」，更不可能讓多個人的持股彙整在同一個視角下比較。
- **共用一份 Google 試算表能解決協作，但解決不了隱私**：最直覺的做法是大家共用一份表格，但這樣等於把所有人的財務細節攤在同一份檔案裡，而且資料通常會落到某一個人（管理者）手上集中保管。
- **市面上的記帳 App 需要使用者各自下載、註冊、學新介面**：對「只是想架一個簡單工具給身邊的人用」這樣的場景太重了。

## 專案亮點

- **零信任管理員的多人協作架構**：每位使用者的試算表活在他自己的 Google Drive，不是集中存在開發者的雲端空間，這是整個專案技術上最核心的挑戰，也是跟一般「大家共用一份雲端表格」做法最大的不同。
- **跨帳戶、跨券商的彙整視角**：券商 App 只能看到自己這家的庫存，這個工具讓使用者可以把多個帳戶（不同券商、不同身份）彙整在同一套損益邏輯下檢視。
- **零設定、零新介面**：不需要下載新 App、不需要註冊新帳號，用已經在用的 LINE 聊天記帳，資料本身又是熟悉的 Google 試算表格式，隨時可以打開來看、自己編輯。
- **完全用免費額度撐起來的工程取捨**：Cloud Run、Cloud Scheduler、Firestore、Google 官方 API 全部刻意限制在免費方案內做架構設計，開發過程想清楚「免費方案的限制是什麼、怎麼繞過去」，而不是遇到限制就只受困或升級權限解決。
- **貼著台股零股交易實際情況設計的記帳邏輯**：換行批次記帳、中文股名模糊比對、移動加權平均成本法、已實現／未實現損益分流、除權息與配股調整——這些不是看盤軟體的強項，也是這個工具真正想做好的部分。

## 核心設計理念

- **隱私優先**：開發者（管理員）不持有、不集中保管任何使用者的財務資料。每位使用者透過自己的 Google 帳號 OAuth 授權，試算表建立在**使用者自己的 Google Drive** 裡，不是集中存在管理員的雲端空間。後端只存一張極小的「連結關係」對照表（LINE 帳號 ↔ 試算表 ID ↔ 加密授權憑證），不存任何實際記帳資料。
- **雙入口都要通**：使用者可以用 LINE 文字記帳，也可以直接打開自己的 Google 試算表手動輸入，兩條路殊途同歸，資料永遠以試算表為準。

## 使用者操作說明

加入 LINE 好友後，使用者將被引導使用 Google 帳號授權、自動在個人的 Google Drive 建立一份記帳試算表(資料庫)，接著就能直接在LINE聊天室記帳。

### 1. 記帳格式（用空白間隔）

| 類型 | 格式 | 範例 |
|---|---|---|
| 買進 | `買 股票 股數 總金額` | `買 台積電 100 85000` |
| 賣出 | `賣 股票 股數 總金額` | `賣 2330 50 48000` |
| 買／賣（不填股數） | `買/賣 股票 總金額` | `買 台積電 85000` |
| 配息 | `配息 股票 金額` | `配息 0050 3000` |
| 配股 | `配股 股票 股數` | `配股 0056 100` |

- 股票可輸入**代碼或中文名稱**，會自動模糊比對（例：`台積電` = `2330`）。
- 單位預設為「股」；輸入 `1張` 會自動換算成 1000 股。
- 可以**一次多行輸入**，一則訊息記多筆，部分行格式有問題只會提示那幾行、不影響其他行。

### 2. 常用指令

| 指令 | 功能 |
|---|---|
| `查詢` | 查看目前持股、損益、總資產 |
| `新增分頁 <名稱>` | 建立一個獨立帳戶分頁（例：`新增分頁 海外帳戶`），分類記錄不同券商／帳戶 |
| `立即同步` | 重新計算試算表損益並寫回狀態欄 |
| `使用說明` | 隨時叫出完整說明 |
| `❌ 刪除上一筆` | 記帳後 5 分鐘內可點按鈕撤回剛剛那筆 |

### 3. 多帳戶記帳

建立多個分頁後，在記帳開頭加上「分頁名稱／」即可指定要記到哪個帳戶：

```
海外帳戶/買 台積電 100 85000
```

若沒加標籤，LINE BOT機器人會用快速按鈕詢問你要記到哪個帳戶。

### 4. 圖表儀表板

輸入「查詢」或由選單點選「查詢」，則可開啟 LIFF 儀表板，檢視持股趨勢、損益走勢與各帳戶／個股的時間序變化（手機開全螢幕，電腦自動升級為寬幅版）。
可先看 [線上 Demo](https://angie-ai1.github.io/StockTool/demo/dashboard_demo.html) 感受呈現效果。

## 技術架構

| 項目 | 選擇 |
|---|---|
| 後端 | Python 3.10+ / FastAPI（大量 I/O 等待操作，適合 async） |
| 資料儲存 | Google Sheets（使用者自己 Drive 裡的試算表）+ OAuth 2.0 |
| 系統對照表 | Google Cloud Firestore（僅存連結關係，不存記帳資料） |
| 部署平台 | Google Cloud Run + Cloud Scheduler（免費額度） |
| 前端 | LINE LIFF（手機）+ RWD 網頁（電腦自動升級寬幅版）、ECharts 圖表 |
| 外部資料源 | 臺灣證券交易所 OpenAPI、證券櫃買中心 OpenAPI |

### 功能範圍

- **LINE 記帳**：文字記帳（買／賣／股息／配股），支援換行批次多筆、股票代碼或中文名稱模糊比對、僅限 1:1 私訊（不處理群組訊息，保護隱私）
- **試算表記帳**：可直接在自己的試算表手動輸入，後端定期同步讀回計算
- **損益引擎**：移動加權平均成本法，已實現／未實現損益分流，賣超防呆，現金股息與股票股利（配股）皆支援
- **LINE 直接查詢**：選單按鈕查看目前庫存、損益、總資產
- **LIFF 儀表板**：登入連結狀態、持股趨勢與損益走勢圖、多帳戶／個股時間序
- **帳號安全**：OAuth 授權加密儲存、失效自動偵測與主動通知重新連結、LIFF 身分驗證採 `id_token`（不依賴網址參數）
- **防呆機制**：刪除上一筆（限時）、批次部分失敗時僅提示有問題的行、輸入不清楚時引導重新輸入

## 專案結構

```
app/
├── routers/        # FastAPI 路由：line_webhook / oauth_callback / liff / tick
├── services/       # 核心邏輯：parser / pnl_engine / sheets_client / oauth_service ...
├── models/         # Pydantic 資料模型
└── db/             # Firestore 連線
tests/              # pytest 測試（164 案例）
docs/demo/          # 內建模擬資料的儀表板 Demo（靜態 HTML）
scripts/            # Google Apps Script 等輔助腳本
```

## 本機開發

```bash
poetry install
poetry run pytest          # 執行測試
poetry run uvicorn app.main:app --reload
```

需設定的環境變數（LINE channel、Google OAuth client、Firestore 憑證、範本試算表 ID 等）見 `app/config.py`。
